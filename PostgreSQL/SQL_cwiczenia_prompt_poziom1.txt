# ZASADY TWORZENIA ĆWICZEŃ:

1. **Progresja trudności:** Zaczynaj od prostych SELECT, stopniowo dodawaj złożoność
2. **Realizm biznesowy:** Każde ćwiczenie ma symulować prawdziwą potrzebę biznesową
3. **Używaj prawdziwych tabel AdventureWorks:** person, sales, production, purchasing, humanresources
4. **Różnorodność:** Mieszaj różne typy zapytań
5. **Uczenie przez praktykę:** Każde ćwiczenie uczy konkretnej umiejętności
6. **Sprawdzalne wyniki:** Podaj informacje typu "Powinieneś otrzymać X wierszy"
7. **Wyjaśniaj DLACZEGO:** Nie tylko JAK, ale dlaczego to rozwiązanie jest dobre

# WAŻNE - RÓŻNICE PostgreSQL vs SQL Server:
- Nazwy schematów są LOWERCASE: sales, person, production, purchasing, humanresources
- Nazwy tabel są LOWERCASE: salesorderheader, product, person itp.
- PostgreSQL używa || do konkatenacji stringów (zamiast +)
- ILIKE do case-insensitive LIKE
- LIMIT zamiast TOP
- PostgreSQL nie ma CONVERT() - użyj CAST() lub ::typ
- Brak nawiasów kwadratowych [] wokół nazw - użyj cudzysłowów "" jeśli potrzeba
- PostgreSQL ma typy: TEXT, BOOLEAN, TIMESTAMPTZ, NUMERIC
- Komentarze: -- (liniowe) lub /* */ (blokowe)

# PRZYKŁADOWE TABELE AdventureWorks DO UŻYCIA:

**sales schema:**
- sales.salesorderheader (zamówienia: salesorderid, orderdate, totaldue, customerid, salespersonid, status)
- sales.salesorderdetail (detale zamówień: productid, orderqty, unitprice, linetotal)
- sales.customer (klienci: customerid, personid, storeid, territoryid)
- sales.salesperson (sprzedawcy: businessentityid, saleslastyear, bonus)

**person schema:**
- person.person (osoby: businessentityid, firstname, lastname, persontype)
- person.address (adresy: addressid, addressline1, city, stateprovinceid)
- person.stateprovince (stany/prowincje: stateprovinceid, name, countryregioncode)

**production schema:**
- production.product (produkty: productid, name, listprice, productnumber, color, productsubcategoryid)
- production.productcategory (kategorie: productcategoryid, name)
- production.productsubcategory (podkategorie: productsubcategoryid, productcategoryid, name)

**purchasing schema:**
- purchasing.purchaseorderheader (zamówienia zakupu)
- purchasing.purchaseorderdetail (detale zamówień zakupu)
